改变颜色空间

Posted on 2023-06-03
  • 学习将图像从一个色彩空间转换到另一个.
  • 创建一个应用程序提取视频中的彩色对象.
  • 学习cv.cvtColor,cv.inRange.

在视频中同时提取红色，蓝色，绿色对象。

首先从BRG空间使用cv.cvtColor()将想要提取的颜色转换到HSV空间,然后设置一定的HSV范围.使用cv.inRange得到该颜色的掩模.对多个掩模进行或运算可以得到想要提取的所有颜色的掩模,然后对视频每一帧使用以上掩模进行与运算得到提取结果.

为什么使用HSV色彩空间?因为HSV色彩空间对色彩的提取更容易,它将色彩分为色调,饱和度,明度三部分,每一部分可以单独调整.而BRG空间无法单独调整饱和度和明度,它的色调,饱和度和明度都会随着BRG三个值的调整而同时改变.

  • HSV 表达彩色图像的方式由三个部分组成：

    • Hue（色调）
    • Saturation（饱和度）
    • Value（明度）

  • HSV的色相范围为[0,179]，饱和度范围为[0,255]，值范围为[0,255]。不同的软件使用不同的规模。因此，如果你要将OpenCV值和它们比较，你需要将这些范围标准化。

  • cv.inRange是OpenCV中的一种图像二值化函数，用于将输入图像中的指定颜色范围内的像素赋值为255（白色），其它像素赋值为0（黑色）。

    该函数的语法格式为：dst = cv.inRange(src, lowerb, upperb[, dst])

    其中，各参数解释如下：

    • src：表示输入图像，必须是一个三通道彩色图像。注意：在HSV空间中，图像的H通道范围是[0, 180]，S和V通道范围是[0, 255]。
    • lowerb，upperb：表示颜色范围的下界和上界。可以是一个标量或一个包含三个元素的数组，分别表示B、G、R通道的下/上界值。
    • dst：可选参数，表示输出的二值图像，大小和类型与输入图像相同。如果未提供该参数，则会自动创建一个和输入图像大小和类型相同的二值图像。

    该函数返回一个二值图像，其中颜色范围内的像素值为255，其它像素为0。

    需要注意的是，在HSV（色相、饱和度、亮度/值）色彩空间中应用该函数时，需要对图像进行颜色空间转换，将BGR图像转化为HSV图像，然后选择合适的H、S、V阈值进行处理。

    例如，如果想要提取一幅图像中的蓝色目标区域，则可以先将BGR图像转化为HSV图像，然后根据H通道（色相）的取值范围提取出蓝色区域，即调用cv.inRange函数并指定lowerb=[90,50,50], upperb=[130,255,255]即可。

cv.threshold是OpenCV中的一个图像处理函数，用于对图像进行阈值化处理。对于一幅灰度图像，可以通过cv.threshold将其转为黑白二值图像或者分割出感兴趣的区域。

可以使用该函数生成的图像作为掩模使用。

该函数的语法格式为： retval, dst = cv.threshold(src, thresh, maxval, type)

其中，参数解释如下：

  • src：待处理的图像，必须为单通道灰度图像。
  • thresh：阈值，当像素值高于或低于此值时，可根据type进行不同的处理。
  • maxval：阈值上限，当type取cv.THRESH_BINARY和cv.THRESH_BINARY_INV时有用。
  • type：阈值化类型，常用的有cv.THRESH_BINARY、cv.THRESH_BINARY_INV、cv.THRESH_TRUNC、cv.THRESH_TOZERO和cv.THRESH_TOZERO_INV等五种类型。

cv.THRESH_BINARY表示超过阈值的部分取maxval（最大值），否则取0（最小值）；cv.THRESH_BINARY_INV表示超过阈值的部分取0，否则取maxval。cv.THRESH_TRUNC表示超过阈值的部分设为阈值，否则不变；cv.THRESH_TOZERO表示超过阈值的部分不变，否则设为0；cv.THRESH_TOZERO_INV与cv.THRESH_TOZERO相反。

该函数返回两个值，第一个为阈值val，第二个为输出图像dst，若不需要阈值val可忽略。

cv.bitwise_and是OpenCV中的一个逐位与运算函数，用于对两个数组进行逐位与运算。通常，该函数被用于图像处理中的掩模操作，即将一个掩模图像与原图像进行逐位与运算，可以实现对指定区域的像素进行处理，而其它区域不受影响。

该函数的语法格式为： dst = cv.bitwise_and(src1, src2[, dst[, mask]])

其中，各参数解释如下：

  • src1，src2：需要进行逐位与运算的两个输入数组。它们的类型必须相同且大小相等。
  • dst：可选参数，表示输出的数组，大小和类型与输入数组相同。如果未提供该参数，则会自动创建一个与输入数组相同大小和类型的数组。
  • mask：可选参数，表示掩模图像，大小和类型与输入数组相同。如果提供了该参数，则只有掩模图像为非零值的位置才会进行逐位与运算，否则忽略该位置。

函数返回一个表示逐位与运算结果的数组。在逐位与运算中，只有当两个操作数的对应位都为1时，结果对应位才为1，否则为0。

一个图像有很多像素点组成,每个像素点都有数值,对图像的基本操作本质上是对数组进行操作。

  • numpy是用与快速数组计算的优化库,简单访问每个像素值并修改将非常缓慢,不建议使用.更好的方法是:访问array.item(),修改array.itemset().

  • matplotlib的三通道顺序是RGB,与opencv不同.

  • 使用cv.copyMakeBorder()函数对原始图像进行了不同方式的边界拓展处理，分别将结果保存到replicate、reflect、reflect101、wrap和constant等5个新的变量中。其中，每个函数的第二到第五个参数分别表示上方、下方、左侧和右侧边界需要拓展的像素数量，第六个参数是拓展方式，可以选择cv.BORDER_REPLICATE、cv.BORDER_REFLECT、cv.BORDER_REFLECT_101、cv.BORDER_WRAP、cv.BORDER_CONSTANT等多种方式，最后一个参数为常数值。

cv.setMouseCallback() 是 OpenCV 库中的一个函数，可以绑定鼠标事件和回调函数。当用户在指定的窗口上进行鼠标操作时，会自动调用指定的回调函数，并把相应的事件和鼠标参数传递给回调函数。

具体来说，cv.setMouseCallback() 函数接受三个参数：

  • 窗口名称：指定要绑定鼠标事件的窗口名称。
  • 回调函数：一个回调函数名，即当用户进行鼠标操作时要调用的函数。
  • 事件类型（可选）：指定要绑定的鼠标事件类型，如果不指定则默认绑定所有事件类型。

OpenCV-Python简介

Posted on 2023-05-29 Edited on 2023-05-30

OpenCV-Python是用于OpenCV的Python API，是原始OpenCV C++实现的Python包装器。

OpenCV-Python结合了OpenCV C++ API和Python语言的最佳特性。

  • 代码与原始C/C++代码一样快（因为它是在后台运行的实际C++代码）
  • 在Python中比C/C++编写代码更容易。

OpenCV-Python利用了Numpy，这是一个高度优化的库，用于使用MATLAB样式的语法进行数值运算。所有OpenCV数组结构都与Numpy数组相互转换。这也使与使用Numpy的其他库的集成变得更加容易（例如SciPy和Matplotlib）。

要使用OpenCV-Python编写优化的代码，必须先明白Numpy。

OpenCV是开放源代码计划（仓库地址）。

  • 在64位计算机中，cv.waitKey()返回的是一个64位的整型数，在使用时需要将其与0xFF按位与运算，以获取低8位的结果。这是因为低8位表示键盘输入的ASCII码值，而高56位通常都是0，与其进行按位与运算后会忽略高位的数值，只保留低8位的值。

    如果不将返回值与0xFF进行按位与运算，则在某些情况下可能会导致程序无法正常退出或者产生其他异常。因此，建议在64位计算机上使用cv.waitKey(0) & 0xFF来获取键盘输入的ASCII码值。

  • 在使用cv2.VideoCapture类读取视频流或者本地视频文件时，需要调用release()方法释放资源。其主要作用是停止捕获视频帧，并且将视频流占用的内存等资源释放掉，以便其他应用程序能够访问它们。
